# Child-safety model: adult-owned accounts, layered topic safety, over-block bias

The app targets ages 4–10 and is designed for the unsupervised-child worst case. Accounts are owned by an adult (parent/teacher); children have no credentials of their own (COPPA-driven, and the only sane model for under-13s). The in-session experience is child-operable (big controls, voice, audio Narration) but safe to hand over.

Topic safety is enforced in three layers, cheapest and most reliable first: (1) curated suggestion chips are safe by construction and are the default path, so most input never needs moderation; (2) a moderation gate classifies any free-text or transcribed-voice **Subject** before accepting it, gently redirecting off-limits subjects rather than erroring; (3) constrained primitive-DSL generation is a backstop — no photorealism or in-image text, so even borderline-allowed subjects render tame. The gate is powered by the same LLM (strong system prompt + a classification call), not a dedicated moderation vendor — layers 1 and 3 already constrain the surface, and young children mostly use chips. A dedicated moderation API can be added later if real-world abuse warrants it; rejected subjects are logged to tune from real data. Throughout, we deliberately err toward over-blocking: a false "let's pick something else" is harmless here, a false allow is a disaster.
