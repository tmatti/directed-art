# Narration is pre-generated premium TTS; voice input is browser-based enhancement

Because the Plan is frozen at confirmation (ADR-0002), every Step's narration text is known before the child steps through it. We therefore synthesize each Step's Narration audio once, at partition time, with a premium cloud TTS voice (warm, kid-friendly), store the resulting audio, and serve it statically. This removes per-page TTS latency and per-play cost, and lets us afford a high-quality voice that matters for pre-readers — at the price of a one-time generation step and stored audio files per Directed Drawing.

Voice *input* (the child speaking chat answers) cannot be pre-generated and is treated as an enhancement over the suggestion chips and typing, not the critical path. For MVP it uses the browser Web Speech API (no backend, but Chrome-centric), behind a clean seam so cloud STT can replace it later without touching callers. Rationale for the asymmetry: a child must be able to *hear* steps (many in the 4–10 range can't read), so Narration output is a hard requirement; a child *speaking* answers is convenience, and the chips/typing fallbacks keep flaky browser STT from being a launch blocker.
