- **Polling re-serializes the full drawing every poll** — false. While a plan is
  `generating`, `unconfirmed_candidate` is nil, so polls ship a light

  appears exactly once, when polling stops.
- **Sessions not invalidated on password change** — false. `user.rb:28-30`

  `active_profiles#update` all target.

  (`generate_directed_drawing_job_test.rb:86`, `assert_raises(RuntimeError)`).

  tens of drawings; `profile_id` is already indexed and the confirmed filter is
  cheap at that scale.
- **Per-artwork `url_for` "N+1"** — inherent to signed-URL generation and cheap;
  attachments are already eager-loaded (`directed_drawings_controller.rb:11`).
- **Retry callback on a deleted plan** — in practice a no-op UPDATE; `discard_on
  ActiveJob::DeserializationError` already covers the deserialize case.
- **React index keys in DrawingCanvas** — the list is static per drawing; no bug.
- **Inertia.js 3.3→3.5 minor bump** — routine dependency maintenance, no
  standalone plan warranted.
- **Verbatim rejected-Subject logging** — by design (ADR-0003, log to tune the
  classifier). Not a finding; revisit only if a data-retention policy is added.
